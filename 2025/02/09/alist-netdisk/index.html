使用Koyeb或Render部署AList搭建资源网站

选择Koyeb或Render

通过下方两个平台免费Web服务的优缺点比较选择自己喜欢的平台（我只用过Koyeb和Render）

Koyeb Render
优点 不会因闲置而自动休眠 可以绑定自己的域名
缺点 不能绑定自己的域名 服务在15 分钟内没有收到请求的话会自动关闭服务，当有新请求时，服务需要约 50 秒才能重新启动。

使用Koyeb部署AList

免费的服务不能直接绑定自己的域名，后面会介绍一种方法可以用自己的域名访问部署好的网站（可是下载链接依然是原来Koyeb提供的域名）

点击下面的按钮可以快捷部署：

关于部署时添加的环境变量的说明, 如下所示:

变量名 值
CDN https://cdn.jsdelivr.net/npm/alist-web@latest/dist （也可以使用其他 CDN）
PORT 5244（不能变）
DB_PORT 3306（不能变）
DB_TYPE sqlite3（也可以使用 mysql 和 postgres。mysql 用于远程数据库）
DB_HOST 此处填写数据库主机地址（对于 SQLite3 DB，请留空）
DB_NAME 此处填写数据库名称（对于 SQLite3 DB，请留空）
DB_USER 此处填写 DB 用户名（对于 SQLite3 DB，请留空）
DB_PASS 此处填写数据库密码（对于 SQLite3 DB，请留空）
DB_TABLE_PREFIX alist_（不能变）
DB_SSL_MODE false（仅当使用 MySQL 或 PostgreSQL DB 时。对于 SQLite3 DB，不需要此环境变量）

按照自己的需求填好后点击部署，等待网站部署完毕

让网站能够使用自己的域名访问

  1. 将AlistGo/alist-web: 🗂️ The front end of Alist V3存储库中的文件下载后，获取编译生成的dist文件夹里的内容（也可以在存储库Releases里面直接下载dist的压缩包）

  2. 修改dist文件夹中index.html文件的内容

    将api的部分修改成Koyeb提供的域名，就像这样

  3. 在Github新建存储库，修改后将dist文件夹里的内容上传到里面

  4. 可以使用Cloudflare Pages或者Vercel连接到存有dist文件夹内容的存储库部署网站

  5. Cloudflare Page和Vercel都可以免费绑定自己的域名，绑定好后就能用自己的域名访问网站了

使用Render部署AList

  1. 新建一个Github存储库

  2. 将AlistGo/alist-render: Deploy alist to render存储库中的文件下载后，上传到刚刚新建的Github存储库

  3. 在Render右上角新建Web服务，连接到刚刚新建的GitHub存储库，部署AList

关于部署时添加的环境变量的说明, 如下所示:

变量名 值
CDN https://cdn.jsdelivr.net/npm/alist-web@latest/dist （也可以使用其他 CDN）
PORT 5244（不能变）
DB_PORT 3306（不能变）
DB_TYPE mysql（也可以使用 sqlite3 和 postgres。mysql 用于远程数据库）
DB_HOST 此处填写数据库主机地址（对于 SQLite3 DB，不需要此环境变量）
DB_NAME 此处填写数据库名称（对于 SQLite3 DB，不需要此环境变量）
DB_USER 此处填写 DB 用户名（对于 SQLite3 DB，不需要此环境变量）
DB_PASS 此处填写数据库密码（对于 SQLite3 DB，不需要此环境变量）
DB_TABLE_PREFIX alist_（不能变）
DB_SSL_MODE false（仅当使用 MySQL 或 PostgreSQL DB 时。对于 SQLite3 DB，不需要此环境变量）

按照自己的需求填好后点击部署，等待网站部署完毕

特别注意：如果用Render部署数据库最好不要用sqlite3，服务在15 分钟内没有收到请求的话会自动关闭服务，当有新请求时，服务需要约 50 秒才能重新启动。由于sqlite3是本地的数据库，重新启动后记录的数据都会消失（从初始状态开始）
